2018-04-07 Git 阅读量

2018年4月2日

情景： 今天早上使用 git pull 命令拉取了最新代码，由于提前没有执行 git log 查看日志，结果上次的提交代码只能在生产环境运行，启动项目报错 W(￣_￣)W

处理：
只要撤销刚才的 git pull 就可以了

首先使用 git log 查看提交日志

使用 git reflog 查看之前的操作日志

然后使用 git reset –hard HEAD@{n} ,（n 是你要退回的引用位置）回退

再次查看日志

成功退回 pull 之前的提交版本 (＾－＾)V
